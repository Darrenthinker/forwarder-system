# 仰度科技货代系统 · 部署指南

## 方案对比

| | Zeabur（推荐） | 宝塔VPS自建 |
|---|---|---|
| 难度 | ⭐ 很简单 | ⭐⭐⭐ 需要操作服务器 |
| 速度（国内访问） | ✅ 快（香港节点） | ✅ 最快（国内机房） |

## 方案A：Zeabur 部署（最推荐，30分钟上线）

### 第一步：推送代码到 GitHub
```bash
# 在项目目录执行
git init
git add .
git commit -m "初始化项目"
git remote add origin https://github.com/你的用户名/forwarder-system.git
git push -u origin main
```

### 第二步：在 Zeabur 创建项目
1. 打开 https://zeabur.com，用 GitHub 账号登录
2. 点击 **New Project** → **Deploy from GitHub**
3. 选择刚推送的仓库
4. 选择服务器地区：**Hong Kong（香港）**

### 第三步：添加 PostgreSQL 数据库
1. 在项目里点击 **Add Service** → **Prebuilt** → **PostgreSQL**
2. 创建后点击 PostgreSQL 服务 → **Connect** → 复制 `Connection String`

### 第四步：配置环境变量
在 Next.js 服务的 **Variables** 里添加：
```
DATABASE_URL=（粘贴刚才复制的 PostgreSQL 连接地址）
```

### 第五步：触发重新部署
环境变量保存后 Zeabur 会自动重新部署，等待约2分钟即可。

### 第六步：绑定自定义域名
1. Next.js 服务 → **Domain** → **Add Domain**
2. 输入你的域名，如：`system.yiangdu.com`
3. 按提示在域名商处添加 CNAME 记录指向 Zeabur
4. 等待 DNS 生效（通常5-15分钟）

**朋友的域名**：朋友在自己的域名商处，把他的域名 CNAME 指向同一个 Zeabur 地址即可，两个域名都能访问同一套系统。

### 以后更新
```bash
# 本地开发完，推送代码即自动部署
git add .
git commit -m "更新功能"
git push
# Zeabur 自动检测到 push，2分钟内完成更新
```

## 方案B：宝塔面板 + GitHub Actions（朋友服务器）

### 服务器要求
- 系统：Ubuntu 20.04 / 22.04（推荐）
- 内存：≥ 2GB
- Node.js 20+、PM2、PostgreSQL（或阿里云RDS）

### 第一步：服务器初始化
SSH 登录服务器后执行：
```bash
# 安装 Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt-get install -y nodejs

# 安装 pnpm 和 PM2
corepack enable
corepack prepare pnpm@latest --activate
npm install -g pm2

# 创建项目目录
mkdir -p /www/wwwroot/forwarder-system

# 克隆代码
cd /www/wwwroot/forwarder-system
git clone https://github.com/你的用户名/forwarder-system.git .
```

### 第二步：配置 PostgreSQL
**方式一：用阿里云RDS**（推荐，有自动备份）
- 创建 RDS PostgreSQL 实例
- 获取连接地址

**方式二：本机安装**
```bash
apt-get install -y postgresql
sudo -u postgres psql -c "CREATE USER forwarder WITH PASSWORD '你的密码';"
sudo -u postgres psql -c "CREATE DATABASE forwarder OWNER forwarder;"
```

### 第三步：配置环境变量
```bash
cat > /www/wwwroot/forwarder-system/.env << 'EOF'
DATABASE_URL=postgresql://forwarder:你的密码@localhost:5432/forwarder
NODE_ENV=production
PORT=6419
EOF
```

### 第四步：首次部署
```bash
cd /www/wwwroot/forwarder-system
pnpm install
# 把 schema 切换为 postgresql
sed -i 's/provider = "sqlite"/provider = "postgresql"/' prisma/schema.prisma
pnpm prisma generate
pnpm prisma migrate deploy
pnpm build

# 启动服务
pm2 start .next/standalone/server.js --name forwarder-system
pm2 save
pm2 startup  # 开机自启
```

### 第五步：宝塔配置反向代理
1. 宝塔 → 网站 → 添加站点（域名填你的域名）
2. 设置 → 反向代理 → 目标URL：`http://127.0.0.1:6419`
3. SSL → Let's Encrypt 申请免费证书

### 第六步：配置自动部署（GitHub Actions）
1. GitHub 仓库 → Settings → Secrets → New repository secret，添加：
   - `SERVER_HOST`：服务器IP
   - `SERVER_USER`：`root`
   - `SERVER_SSH_KEY`：服务器SSH私钥（`cat ~/.ssh/id_rsa`）
   - `DATABASE_URL`：PostgreSQL 连接地址

2. 以后 `git push main` 即自动部署到服务器

### 配置自动备份（宝塔计划任务）
宝塔 → 计划任务 → 添加：
- 类型：Shell脚本
- 执行周期：每天 02:00
- 脚本内容：
```bash
source /www/wwwroot/forwarder-system/.env
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p /www/backup/forwarder
pg_dump "$DATABASE_URL" | gzip > /www/backup/forwarder/db_$DATE.sql.gz
find /www/backup/forwarder -name "*.sql.gz" -mtime +30 -delete
echo "备份完成：db_$DATE.sql.gz"
```

## 数据恢复
```bash
# 查看备份列表
ls /www/backup/forwarder/

# 恢复指定备份
source /www/wwwroot/forwarder-system/.env
gunzip -c /www/backup/forwarder/db_20260419_020000.sql.gz | psql "$DATABASE_URL"
```

## 朋友如何用自己的域名访问
**同一套系统，多个域名**：
- 你的域名：`system.yourdomain.com` → 指向同一服务器/Zeabur
- 朋友的域名：`logistics.friends-domain.com` → 指向同一服务器/Zeabur
- 两个域名访问的是同一个系统，数据完全共享

在宝塔里：网站 → 你的站点 → 域名管理 → 添加朋友的域名即可。
